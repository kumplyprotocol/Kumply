# KUMPLY Compliance L1 — Architecture & Roadmap

> **Status:** ✅ Live on Fuji Testnet (deployed 2026-05-17)
> **Last updated:** 2026-05-17

A dedicated Avalanche L1 where **only KYB-verified institutions can validate**. Every block is signed by a licensed entity — compliance becomes a property of the consensus layer itself.

This document captures the full architectural rationale, technical details, and deployment path. The corresponding code lives in [`contracts/`](./contracts/) and is fully tested.

---

## 1. Why a dedicated L1?

KUMPLY already works on the Avalanche C-Chain with the `AttestationStore` + `ComplianceGate` pattern. So why build a separate L1?

| Property | C-Chain today | Compliance L1 |
|---|---|---|
| **Who validates** | Anyone with 2000 AVAX (legacy) / dynamic fee (post-ACP-77) | **Only Tier-4 (KYB) verified institutions** |
| **Gas cost** | Shared pool, peak gas spikes affect compliance reads | Predictable, KMP gas token, near-zero for compliance lookups |
| **Throughput** | Shared with everything on C-Chain | Reserved exclusively for compliance workloads |
| **Regulatory story** | "We deployed contracts to Avalanche" | "Every block on this chain is signed by a licensed institution" |
| **Cross-L1 reach** | Reads must originate from C-Chain or use ICM | Native ICM hub — every Avalanche L1 trusts KUMPLY attestations directly |

The L1 is **not a replacement** for C-Chain attestations — it is a **specialized network** where regulatory clarity is built into the consensus rules.

## 2. Validator Architecture (ACP-77 + ACP-99)

The KUMPLY L1 follows the latest Avalanche standards:

- **[ACP-77 — Reinventing Subnets](https://build.avax.network/docs/acps/77-reinventing-subnets):** Validators do **not** stake 2000 AVAX. They pay a continuous P-Chain fee proportional to active validator count. This makes institutional validation economical.
- **[ACP-99 — ValidatorSetManager Contract](https://build.avax.network/docs/acps/99-validatorsetmanager-contract):** A Solidity contract on C-Chain manages the canonical validator set. Changes are sent to the P-Chain via Avalanche Warp Messages.

### `KumplyValidatorSetManager.sol`

Our custom implementation lives at [`contracts/contracts/KumplyValidatorSetManager.sol`](./contracts/contracts/KumplyValidatorSetManager.sol). It extends the ACP-99 spec with these KUMPLY-specific properties:

1. **KYB gate.** `initializeValidatorRegistration()` calls `AttestationStore.verify(msg.sender)` and reverts if `tier < 4`.
2. **Self-healing.** `disableExpiredValidator(owner)` is permissionless — any EOA can purge a validator whose KYB attestation has expired. No admin keys required.
3. **Anti-centralization caps.** Each validator's weight is capped at 20% (`MAX_VALIDATOR_WEIGHT_BPS = 2000`) of the total active weight.
4. **Churn limits.** Max 20 weight-changes per 24h epoch (`MAX_CHURN_PER_EPOCH`) — prevents flash takeovers.
5. **Warp-compatible events.** `ValidatorRegistered`, `ValidatorRemoved`, `ValidatorWeightUpdated` emit Warp-compatible payloads that the P-Chain consumes.

### Validator lifecycle

```
                ┌─────────────────────────────┐
                │  KYB-verified institution   │
                │  (banks, funds, consortia)  │
                └──────────────┬──────────────┘
                               │ initializeValidatorRegistration(nodeID, weight)
                               ▼
                ┌─────────────────────────────┐
                │  KumplyValidatorSetManager  │
                │  - reads AttestationStore   │
                │  - enforces tier >= 4       │
                │  - emits Warp event         │
                └──────────────┬──────────────┘
                               │ Warp Message (ACP-77)
                               ▼
                ┌─────────────────────────────┐
                │           P-Chain           │
                │  canonical L1 validator set │
                └──────────────┬──────────────┘
                               │ Snowman++ consensus
                               ▼
                ┌─────────────────────────────┐
                │      KUMPLY L1 (live)       │
                │  Subnet-EVM · 2s blocks     │
                └─────────────────────────────┘
```
